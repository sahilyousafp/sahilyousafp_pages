CityMind LLM Urban Agents

A comprehensive modelling of human behaviour in an urban environment. Each of 500 agents reasons about movement using an LLM — perceiving street-level imagery, maintaining memory, and adapting based on needs, personality, and emotional state. Set in Barcelona's Eixample district.

Research Question+

Can urban ABM simulations be capable of behavioural spatial awareness and action?

Multi-agent simulation — Barcelona Eixample

CityMind is an agent-based pedestrian simulation where each agent reasons about its movement decisions using a large language model instead of following predefined rules. Agents perceive their surroundings through street-level imagery, maintain memory of visited places, and adapt their behavior based on individual needs, personality, and emotional state.

The project addresses gaps in prior work: Generative Agents (Simulacra) achieved 85% similarity to real human behavior but used planar virtual environments. CitySim simulated movement in Shibuya but lacked traceable paths and spatial awareness. CityMind combines traceable spatial movement on real urban networks with street-level visual perception and emotionally-aware LLM cognition.

The Agent+

Profile, mobility, emotions, LLM selection

Character selection — 4 archetypes
Hybrid LLM + rule-based mobility
Single Agent Lab — emotion mix, cognition, perception

Each agent has a profile (personality, preferences, age, background) and a daily plan structured into time blocks driving goal selection. The mobility system uses a hybrid LLM + rule-based approach with four decision blocks per step: Needs, Cognition, Plan, Mobility.

Emotions operate on two timescales: Needs Module (hunger, energy, social, comfort — updated per step) and Emotion Module (excited, stressed, relaxed, bored — updated every 10 steps via Russell's circumplex model).

LLM selection guided by EQ-Bench V2 Emotional Intelligence Score: DeepSeek V4 Fast (82.6) chosen for best emotional reasoning.

Prompt Architecture+

Cognition, mobility, needs prompts

Cognition Prompt: Produces vivid inner monologues grounded in specific events. Anti-cliché rules forbid filler phrases. Needs-to-mood mapping mirrors psychology: hunger > 0.7 → stressed; energy < 0.3 → bored; unmet social → disengaged.

Mobility Prompt: Gives the LLM rich sensory context — scene descriptions, candidate street edges with [NEW]/[visited] tags, distance-aware urgency levels (FREE EXPLORATION → GETTING CLOSE → ALMOST THERE). Archetype-specific perception guidance shapes what the agent notices.

Needs Prompt: Two LLM evaluation paths — Visual Satisfaction (every 5 steps, calibrated +0.06 max for excellent spaces) and Amenity Satisfaction (within 30m, considers agent's mental state). Rule-based fallback for graceful degradation.

Human Calibration+

Needs decay, cross-need coupling, exploration budgets

Every parameter calibrated against real-world data. Each simulation step = 15 minutes (96 steps/day). Cross-need coupling models human physiology: hunger → energy loss (×1.3), discomfort → fatigue (×1.2), exhaustion → amplified hunger (×1.2).
